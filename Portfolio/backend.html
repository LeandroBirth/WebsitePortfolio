ForumHUB - REST API

Completed during the ONE (Oracle Next Education) program, hosted on the Alura platform.

Overview

This project is a REST API for managing a forum where users can:

  • Create, view, update, and delete topics.
  • Add and manage responses to topics.
  • Authenticate and authorize users with JWT.

The API is built using Spring Framework and adheres to REST best practices with proper validations, database integration, and secure access control.

Main Endpoints

Method URL Description
[GET] /topics Lists all topics.
[GET] /topics/{id} Retrieves a specific topic.
[POST] /topics Creates a new topic.
[PUT] /topics/{id} Updates an existing topic.
[DELETE] /topics/{id} Deletes a topic.
[GET] topics/{id}/answers Lists all responses to a topic.
[POST] /topics/{id}/answers Adds a response to a topic.
[POST] /answers/{id}/answer_solution Marks a response as the topic solution.
[POST] /api/login Logs in to receive a JWT token.
[POST] /api/register Registers a new user.

Technologies Used

  • Language: Java
  • IDE: IntelliJ IDEA
  • Framework: SpringBoot
  • Spring Data JPA, Spring Boot Starter Web, MySQL, Flyway, Lombok, Spring Validation, Spring Boot Security, Java-JWT.

Calculator for Android (JAVA): Birth Calculator +-

Description: Birth Calculator +- is an Android-based calculator app with a user-friendly interface designed to handle basic arithmetic operations such as addition, subtraction, multiplication, and division. Built as part of a project for Vanderbilt University's Java for Android program, this app was developed in Java using Android Studio and demonstrates fundamental concepts of Android app development.

Key Features:

  • Basic Operations: Supports addition, subtraction, multiplication, and division.
  • Decimal Support: Updated to accept decimal inputs, making it suitable for operations requiring floating-point precision.
  • Error Handling: Displays appropriate messages when invalid inputs are detected, such as division by zero.

Project Structure:

  • Logic Layer: Processes user inputs and performs the chosen arithmetic operation.
  • UI Layer: Facilitates user interaction and displays results.

Code Improvements:

  1. Double Data Type Integration: The calculator initially accepted only integer inputs. It was modified to handle decimal values (e.g., 3.14, 0.75).
  2. User Interface Enhancements:
    • Decimal Input Support: Updated EditText input fields to accept decimal values, ensuring compatibility with the new double functionality.
